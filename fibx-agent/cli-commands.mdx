All commands follow the pattern:

```bash
npx fibx@latest <command> [arguments] [--chain <chain>] [--json]
```

Default chain is `base` unless overridden with `--chain`.

---

## auth — Authentication

Manage the wallet session. Required before any on-chain operation.

### auth import

Import an existing private key to create a local wallet session. This is the most direct way to authenticate — if you already have a wallet, use this method. The CLI opens an interactive prompt where you paste the key.

```bash
npx fibx@latest auth import
```

The key is encrypted and stored in a local session file. No external server holds your private key with this method.

### auth login / auth verify (Email OTP)

Alternative authentication via email OTP. This creates a server-managed wallet through [Privy](https://privy.io) — useful when you do not have an existing private key.

```bash
# Step 1: Send OTP to email
npx fibx@latest auth login <email>

# Step 2: Verify with the code received
npx fibx@latest auth verify <email> <code>
```

| Parameter | Type   | Required | Description                 |
| --------- | ------ | -------- | --------------------------- |
| `email`   | string | Yes      | User's email address        |
| `code`    | string | Yes      | OTP code received via email |

### auth logout

End the active session.

```bash
npx fibx@latest auth logout
```

## send — Transfer Tokens

Send native tokens or ERC-20 tokens to a recipient address. The transaction is simulated before execution.

```bash
npx fibx@latest send <amount> <recipient> [token] [--chain <chain>] [--json]
```

| Parameter   | Type   | Required | Description                                  |
| ----------- | ------ | -------- | -------------------------------------------- |
| `amount`    | number | Yes      | Amount to send                               |
| `recipient` | string | Yes      | Destination address (`0x...`)                |
| `token`     | string | No       | Token symbol (e.g. `USDC`). Omit for native token |
| `chain`     | string | No       | `base`, `citrea`, `hyperevm`, or `monad`     |
| `json`      | flag   | No       | Output as JSON                               |

If `token` is omitted, the chain's native token is used (ETH on Base, cBTC on Citrea, HYPE on HyperEVM, MON on Monad).

### Examples

```bash
# Send native ETH on Base
npx fibx@latest send 0.01 0xRecipient

# Send ERC-20 on Base
npx fibx@latest send 100 0xRecipient USDC

# Send native MON on Monad
npx fibx@latest send 1 0xRecipient --chain monad
```

## trade — Swap Tokens

Exchange one token for another via Fibrous aggregation. The CLI finds the optimal route, handles token approvals, simulates the swap, and executes.

```bash
npx fibx@latest trade <amount> <from_token> <to_token> [--chain <chain>] [--slippage <n>] [--approve-max] [--json]
```

| Parameter     | Type   | Required | Description                                    |
| ------------- | ------ | -------- | ---------------------------------------------- |
| `amount`      | number | Yes      | Amount of the source token to swap             |
| `from_token`  | string | Yes      | Source token symbol (e.g. `ETH`, `USDC`)       |
| `to_token`    | string | Yes      | Target token symbol (e.g. `USDC`, `DAI`)       |
| `chain`       | string | No       | `base`, `citrea`, `hyperevm`, or `monad`       |
| `slippage`    | number | No       | Slippage tolerance in percent (default: `0.5`) |
| `approve-max` | flag   | No       | Use infinite approval instead of exact amount  |
| `json`        | flag   | No       | Output as JSON                                 |

<Note>
  The `trade` command automatically detects **Wrap** (Native → Wrapped) and **Unwrap** (Wrapped → Native) operations and executes them directly via contract calls, bypassing the aggregator routing to save gas.
</Note>

### Examples

<Tabs>
  <Tab title="Base">
    ```bash
    npx fibx@latest trade 0.1 ETH USDC
    ```
  </Tab>
  <Tab title="Citrea">
    ```bash
    npx fibx@latest trade 0.01 cBTC USDC --chain citrea
    ```
  </Tab>
  <Tab title="Monad">
    ```bash
    npx fibx@latest trade 1 MON USDC --chain monad
    ```
  </Tab>
  <Tab title="Custom Slippage">
    ```bash
    npx fibx@latest trade 0.1 ETH USDC --slippage 1.0
    ```
  </Tab>
</Tabs>

## aave — Aave V3 (Base Only)

Manage Aave V3 lending positions on Base. Supply assets to earn yield, borrow against collateral, repay debt, or withdraw.

<Warning>
  Aave V3 operations are only available on **Base**. Attempting Aave commands on other chains will fail.
</Warning>

```bash
npx fibx@latest aave <action> [amount] [token] [--json]
```

| Parameter | Type   | Required              | Description                                       |
| --------- | ------ | --------------------- | ------------------------------------------------- |
| `action`  | string | Yes                   | `status`, `supply`, `borrow`, `repay`, `withdraw` |
| `amount`  | string | Yes (except `status`) | Amount or `max` for full repay/withdraw           |
| `token`   | string | Yes (except `status`) | Token symbol (`USDC`, `ETH`, `DAI`, etc.)         |
| `json`    | flag   | No                    | Output as JSON                                    |

### Actions

| Action     | Description                                | Example                                          |
| ---------- | ------------------------------------------ | ------------------------------------------------ |
| `status`   | Account health, LTV, net worth             | `npx fibx@latest aave status`                   |
| `supply`   | Deposit assets (auto-wraps ETH → WETH)     | `npx fibx@latest aave supply 1 ETH`             |
| `borrow`   | Borrow against collateral                  | `npx fibx@latest aave borrow 50 USDC`           |
| `repay`    | Repay debt (auto-wraps ETH → WETH)         | `npx fibx@latest aave repay max ETH`            |
| `withdraw` | Withdraw assets (auto-unwraps WETH → ETH)  | `npx fibx@latest aave withdraw max ETH`         |

### ETH ↔ WETH Auto-Wrap

`supply`, `repay`, and `withdraw` support automatic ETH ↔ WETH conversion. Use `ETH` as the token symbol — the CLI handles wrapping and unwrapping internally.

```bash
npx fibx@latest aave supply 1 ETH         # Auto-wraps ETH → WETH and supplies
npx fibx@latest aave supply 100 USDC      # Supply ERC-20 directly
npx fibx@latest aave borrow 50 USDC       # Borrow
npx fibx@latest aave repay 50 USDC        # Repay partial USDC debt
npx fibx@latest aave repay max ETH        # Auto-wraps ETH and repays full WETH debt
npx fibx@latest aave withdraw max ETH     # Withdraws WETH and auto-unwraps to ETH
```

### Health Factor

Before borrowing, always check `aave status`. The Health Factor indicates liquidation risk:

- **Above 1.5** — Safe to proceed
- **Between 1.1 and 1.5** — Liquidation risk warning
- **Below 1.1** — High liquidation risk, proceed with extreme caution

<Note>
  When fully closing a position, always use `max` as the amount. This sends `MAX_UINT256` to the Aave contract and prevents dust residuals from remaining.
</Note>

## config — RPC Configuration

View and modify local fibx CLI configuration. Used primarily to set custom RPC URLs when hitting rate limits on public endpoints.

```bash
npx fibx@latest config <subcommand> [chain] [url]
```

| Subcommand  | Description                             | Example                                                                  |
| ----------- | --------------------------------------- | ------------------------------------------------------------------------ |
| `set-rpc`   | Set a custom RPC URL for a chain        | `npx fibx@latest config set-rpc base https://mainnet.base.org`          |
| `get-rpc`   | View the current RPC URL for a chain    | `npx fibx@latest config get-rpc base`                                   |
| `list`      | List all custom configurations          | `npx fibx@latest config list`                                            |

| Parameter | Type   | Required | Description                                  |
| --------- | ------ | -------- | -------------------------------------------- |
| `chain`   | string | Yes      | `base`, `citrea`, `hyperevm`, or `monad`     |
| `url`     | string | Yes (`set-rpc`) | Full HTTP(S) RPC endpoint URL         |

<Note>
  Configuration is stored locally at `~/.config/fibx/config.json` and persists across sessions.
</Note>

### Examples

```bash
# Handle rate limit on Base
npx fibx@latest config set-rpc base https://mainnet.base.org

# Check configured RPC
npx fibx@latest config get-rpc base

# List all configurations
npx fibx@latest config list
```

## tx-status — Transaction Status

Check the on-chain status of a transaction and retrieve the block explorer link.

```bash
npx fibx@latest tx-status <hash> [--chain <chain>] [--json]
```

| Parameter | Type   | Required | Description                                  |
| --------- | ------ | -------- | -------------------------------------------- |
| `hash`    | string | Yes      | Transaction hash (`0x...`)                   |
| `chain`   | string | No       | Must match the chain of the original transaction |
| `json`    | flag   | No       | Output as JSON                               |

<Note>
  Always use the same `--chain` flag that was used for the original transaction. A Base transaction hash will not be found on Monad.
</Note>

### JSON Output

```json
{
  "status": "success",
  "blockNumber": "12345",
  "gasUsed": "21000",
  "from": "0x...",
  "to": "0x...",
  "explorerLink": "https://basescan.org/tx/0x...",
  "chain": "base"
}
```

## Error Reference

Common errors across all commands:

| Error                  | Cause                                          | Resolution                                        |
| ---------------------- | ---------------------------------------------- | ------------------------------------------------- |
| `Not authenticated`    | No active session                              | Run `auth login` or `auth import` first           |
| `Insufficient balance` | Not enough tokens for the operation             | Check `balance` and adjust the amount              |
| `Simulation failed`    | Transaction would revert on-chain               | Verify parameters and token availability           |
| `No route found`       | No liquidity path exists for the token pair     | Try a different pair or check the chain             |
| `Slippage exceeded`    | Price moved beyond tolerance                    | Retry or increase slippage with `--slippage`        |
| `Session expired`      | Authentication session timed out                | Re-authenticate with `auth login`                   |
| `Invalid address`      | Recipient address is malformed                  | Verify the address starts with `0x` and is valid    |
| `Rate limit`           | Too many requests in a short period             | Wait 60 seconds and retry                           |
